## 4. 발견된 잠재 이슈 (3건)

### 4.1 이슈 #1 — v0.3 기획안의 "PG 16"과 D-3의 "PG 17" 불일치

- **위치**: `AI컨텐츠허브_기획안_v0.3.md` §10.2 vs `00_확정사항_레지스트리.md` D-3
- **영향**: 낮음 (개발 시 PG 17만 사용)
- **권장**: v0.3 기획안 한 줄 메모 추가 또는 v0.4로 갱신 시점에 통일

### 4.2 이슈 #2 — "이미지 50장/일" 가정의 데스크탑 부하

- A3 §10에 "1장당 30~60초"라 했는데 일 50장은 25~50분의 GPU 시간
- AMD R9700이 처음 가동되는 모델이라 실제 성능 데이터가 없음. Phase 0 §H.5에서 베이스라인 측정 후 A3 §10 갱신 필요
- **영향**: 중간 (Phase 1 §1.2 완료 조건 영향 가능)
- **권장**: Phase 0 W3 PoC에서 성능 측정값을 A3·Phase 1에 반영

### 4.3 이슈 #3 — Auth.js v5 + FastAPI 세션 통합의 어댑터 미정

- A2 §1.5는 "Next.js Auth.js v5 → DB sessions 테이블 → FastAPI가 쿠키로 검증"이라 명시
- 하지만 Auth.js의 sessions 테이블 스키마는 자체 `session_token`·`expires`·`userId` 형식. A1 §3.4 컬럼은 호환되나 **마이그레이션 시 정확 일치 검증 필요**
- **영향**: 중간 (Phase 1 §2.W1 영향)
- **권장**: Phase 1 W1에 Auth.js v5 PG adapter docs 1회 따라 해보고 schema 일치 검증, 필요시 A1 §3.4 미세 조정

## 7. 위험 요소 종합 (Phase별 합계)

가장 큰 위험 5가지 (모든 Phase 횡단):

1. **본업 병행 시간 부족** — 모든 Phase의 가능성 高 → 자동화 우선 설계로 대응 (현 기획서가 그대로 그 방향)
2. **AdSense 승인 거절** — Phase 3 가능성 中 → 30편+ 사전 콘텐츠, 정책 100% 준수
3. **데스크탑 ComfyUI 불안정** — Phase 1+ 가능성 中 → 외부 API 폴백
4. **콘텐츠 표절·라이선스 분쟁** — 가능성 低·영향 高 → A5 검토 + AI 명시
5. **1인 운영 번아웃** — 가능성 中·영향 高 → 주 5~6시간 캡, 자동화 80%+ 목표
