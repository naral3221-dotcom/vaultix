## 0. 사용 방법

### 0.1 시드 SQL 생성

본 문서의 표를 SQL INSERT로 변환하는 스크립트:

```python
# scripts/seed_prompts.py — Phase 1 W6에 작성
from vaultix_api.db.session import SessionLocal
from vaultix_api.models.prompt_template import PromptTemplate
import json

SEEDS = [
    # 아래 표를 dict 리스트로 옮긴 것
    {"name": "biz_meeting_modern", "asset_type": "image", "category_slug": "business",
     "base_prompt": "modern business meeting illustration, {style}, {mood}, ...",
     "negative_prompt": "blurry, low quality, watermark, text",
     "variables_json": {"style": ["flat", "isometric", "outline"],
                        "mood": ["focused", "collaborative", "energetic"]},
     "aspect_ratio": "1:1", "preferred_model": "flux-dev",
     "workflow_template": "flux_dev_basic_1024.json", "weight": 100},
    # ...
]

with SessionLocal() as db:
    for s in SEEDS:
        cat = db.execute(...).scalar()  # category_slug → category_id
        db.add(PromptTemplate(...s, category_id=cat.id))
    db.commit()
```

### 0.2 변수 치환 규칙

`base_prompt`의 `{key}` 자리에 `variables_json[key]`에서 랜덤 1개를 뽑아 치환. 한 템플릿이 변수 N개면 조합 수 = 모든 키 길이의 곱.

> 예: `{style}` 3개 × `{mood}` 3개 = 9가지 변형. 각 변형마다 seed 다르게 → 일 1템플릿당 5~10장 생성 가능.

### 0.3 가중치 운영

- 신규 템플릿: weight=100
- 다운로드 평균이 카테고리 평균보다 1.5배 높으면 weight +20
- 1.5배 낮으면 weight -20
- weight=0이면 자동 생성 큐에서 제외 (인박스 수동 사용 가능)

## 6. 시드 운영 룰

### 6.1 신규 템플릿 추가 빈도

- 주 5~10개 신규 추가 (Phase 2~3 기간)
- 어드민에서 직접 입력 또는 LLM에게 "다음 카테고리에 어울리는 프롬프트 템플릿 5개 제안"

### 6.2 비활성화 룰

- 30일간 발행된 자산의 평균 다운로드 < 5 → weight 50으로 감소
- 60일간 같은 기준 → weight 0 (자동 발행 중단, 어드민에서 수동 사용 가능)

### 6.3 시즌성 자동 부스트 (Phase 4)

- 봄/여름/가을/겨울 카테고리: 해당 계절 1개월 전부터 weight x2
- 명절(설/추석): 2주 전부터 weight x3
- 졸업/입학 시즌: 1월·2월·8월 weight x2 (이력서·자소서)
- 연말: 11월·12월 weight x2 (연말결산·OKR)

## 7. 인박스 카테고리 분류 신뢰도 향상

### 7.1 Phase 1 (단순)

- prompt_template.category_id를 그대로 asset.category_id로 매핑
- LLM 메타데이터 생성 시 카테고리 추천도 받음 (검증용)

### 7.2 Phase 2 (임베딩)

- CLIP 임베딩 + 카테고리 centroid 거리로 자동 분류
- 신뢰도 < 0.6이면 어드민 수동 결정 필요
