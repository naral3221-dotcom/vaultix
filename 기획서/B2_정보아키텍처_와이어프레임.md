# B2 — 정보 아키텍처 & 와이어프레임

> **목적**: 사용자가 사이트에 들어와서 어떤 경로로 다운로드/즐겨찾기/회원가입에 도달하는지를 페르소나별로 명시하고, 핵심 6개 페이지의 레이아웃을 텍스트 와이어로 고정한다.
> **작성일**: 2026-04-27
> **선행**: B1 브랜드·디자인시스템 / 00_확정사항_레지스트리 §5

## 1. 페르소나별 동선

### 1.1 1순위 — 박지원(직장인) — "월요일 아침 보고서용 그래프 이미지 1장"

```
[검색엔진]
   ↓ "비즈니스 미팅 일러스트" (오가닉 검색)
[자산 상세 페이지] (랜딩, AI 생성 배지·라이선스 요약 노출)
   ↓ [다운로드] 클릭
[회원가입 모달] (1분이면 끝, 영구 사용권 설명)
   ↓ 이메일 + 비밀번호
[이메일 인증 안내]
   ↓ 메일 확인 → 인증 링크 클릭
[자산 상세 페이지] (자동 복귀 + 다운로드 자동 시작)
[Toast] "다운로드 완료. 잘 쓰셨길 바라요."
[관련 자산 그리드] (재방문 유도)
```

**KPI**:
- 랜딩 → 다운로드 완료까지 ≤ 5분 (이메일 인증 포함), 5클릭 이내
- 첫 다운로드 후 7일 내 재방문율 30%+

### 1.2 2순위 — 김보미(디자이너·마케터) — "이번 주 인스타 콘텐츠 영감"

```
[직접 방문 또는 Pinterest 핀]
   ↓
[홈] (큐레이션 컬렉션 노출 — "이번 주 트렌드", "신규 100선")
   ↓ 컬렉션 클릭
[탐색 /explore?type=image&category=trend]
   ↓ 마소닉 그리드 무한 스크롤, 호버하며 둘러봄
   ↓ 마음에 드는 카드 ♡
[자산 상세] (확대 미리보기, "이 스타일 더 보기" 클릭)
   ↓ 비슷한 자산 그리드
[다운로드 5장 연속] (시간당 30회 한도 안내 노출)
[블로그 — "이 스타일 어떻게 만들었나" 링크]
   ↓
[블로그 글] (워크플로우 글)
[뉴스레터 구독 인박스 노출]
```

**KPI**:
- 평균 세션 페이지 5+
- 즐겨찾기 사용률 25%+
- 블로그 → 자산으로 역방향 유입 15%+

### 1.3 3순위 — 이준호(취준생) — "신입 직무 이력서 템플릿"

```
[검색엔진] "신입 마케팅 이력서 템플릿"
   ↓
[블로그 — "2026년 신입 이력서 트렌드"]
   ↓ 글 안 "추천 템플릿" 인라인 카드
[자산 상세 — DOCX 이력서]
   ↓ [다운로드] 클릭
[회원가입]
   ↓
[다운로드 + Toast]
[관련 자산: 자기소개서·포트폴리오 PDF·면접 PPT 추천]
```

**KPI**:
- 블로그 → 자산 다운로드 전환 8%+
- 시즌(9~12월·3~5월) MAU 50% 상승

## 3. 글로벌 레이아웃

### 3.1 헤더 (모든 페이지 공통)

```
┌────────────────────────────────────────────────────────────────────────┐
│ [LOGO]   탐색 ▾   블로그   가이드   프롬프트  │ [🔍 검색]  [☀️/🌙] [♡] [👤▾]│
└────────────────────────────────────────────────────────────────────────┘
```

- LOGO: 워드마크 (Phase 1 임시)
- "탐색 ▾": 드롭다운으로 type별 빠른 진입 (이미지/PPT/SVG/DOCX/...)
- 검색: 80자 이내, Enter → /search
- 다크모드 토글
- ♡: 즐겨찾기 (비로그인 시 모달 안내)
- 👤▾: 로그인 상태면 사용자 메뉴, 비로그인이면 [로그인] [회원가입] 버튼

### 3.2 모바일 헤더

```
┌──────────────────────────────────────┐
│ [≡]  [LOGO]            [🔍] [☀️/🌙]│
└──────────────────────────────────────┘
```

- ≡: 좌측 시트로 카테고리/계정/언어 등 펼침
- 검색은 별도 페이지(/search) 진입

### 3.3 푸터

```
┌──────────────────────────────────────────────────────────────┐
│  사이트              컨텐츠            법적 고지              │
```

## 8. 다운로드 흐름 보안·UX 디테일

| 항목 | 결정 |
|------|------|
| signed URL 유효시간 | 5분 (B-04 관련) |
| 동일 URL 재요청 | 1회 (URL 자체에 nonce, 사용 시 폐기) |
| 다운로드 진행 표시 | 브라우저 기본 다운로드 표시 의존 (UX 단순) |
| 다운로드 도중 새로고침 | URL은 invalidated 상태이므로 다시 클릭하면 새 URL 발급 |
| 모바일 다운로드 | iOS Safari 호환 위해 `<a download>` + `Content-Disposition: attachment` |
| 시간당 30회 임계 | 25회 도달 시 [Toast] "5회 남았어요" 1회 안내 |
| 어뷰징 의심 | 5분 내 동일 IP 다른 계정 3개 이상 → CAPTCHA 강화 |

## 부록 A. 첫 발자국 시나리오 (PMF 시뮬레이션)

> "이 흐름이 막히면 사이트가 작동하지 않는다" 5가지 케이스. Phase 1 QA 시나리오로 그대로 사용.

1. **검색엔진 → 자산 상세 → 회원가입 → 인증 → 다운로드** (가장 중요한 단일 흐름)
2. **홈 → 컨텐츠 타입 카드 → 탐색 → 다운로드** (탐색 중심 사용자)
3. **자산 상세 → ♡ → 회원가입 모달 → 즐겨찾기 저장 → 계정 페이지에서 확인** (저장 흐름)
4. **블로그 글 → 인라인 자산 카드 → 자산 상세 → 다운로드** (콘텐츠 마케팅 흐름)
5. **시간당 31번째 다운로드 시도 → 안내 → 1시간 뒤 재시도 → 정상** (한도 운영 흐름)
