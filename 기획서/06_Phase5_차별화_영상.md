# Phase 5 — 차별화 & 영상 + 디지털 상품 + B2B 상세 기획서

> **목적**: AdSense·제휴 외 **다각화 수익(디지털 상품·B2B 라이선스·구독)** + **차별화 컨텐츠(영상·음향)** + **모바일 앱**으로 사이트의 두 번째 성장 곡선을 만든다.
> **선행**: Phase 4 §7 진입 조건 모두 통과 / 손익분기 도달 / 8개 컨텐츠 타입 안정 운영
> **작성일**: 2026-04-27
> **상태**: 본 문서는 **방향성·원칙 + 가능한 옵션 매트릭스** 위주로 작성됩니다. Phase 4 종료 시점의 데이터(어떤 컨텐츠가 잘 팔리는지·B2B 문의가 있었는지·트래픽 규모·본업 상황)에 따라 W1~Wn은 그때 확정합니다.

## 1. 목표 & 전제

### 1.1 목표 (선택형)

Phase 5는 **모든 항목을 다 하지 않습니다**. Phase 4 결과 데이터에 따라 다음 5개 트랙 중 1~3개를 선택해 진행합니다.

| 트랙 | 결정 트리거 |
|------|------------|
| **T1. 영상 컨텐츠 (저비용 합성)** | 인스타·유튜브 쇼츠 트래픽이 의미 있게 잡힘 |
| **T2. 음향·음악** | 영상 트랙과 결합 가치 있고, 라이선스 문제 정리 가능 |
| **T3. 자체 디지털 상품 (프리미엄)** | 무료 자산 다운로드 톱 10이 명확하고 "프리미엄 버전" 수요 신호 있음 |
| **T4. B2B 벌크 라이선스 / API** | 실제 B2B 문의 3건 이상 도착 |
| **T5. 모바일 앱 (PWA → 네이티브)** | MAU 30,000+ 달성, 모바일 비중 50%+ |

### 1.2 전제

- 본업 시간 변동 가능성 — 본업 비중이 늘어 시간이 줄면 Phase 5는 자동화 우선 트랙(T3, T4)에 집중
- 본업 시간이 줄어 사이드가 메인이 될 가능성 — 그때 T5(모바일 앱) 검토

## 2. 트랙 1 — 영상 컨텐츠 (저비용 합성)

### 2.1 원칙 (T-02 정책 유지)

- **영상 생성 API 사용 X** (Runway/Pika/Sora 비용 폭탄)
- **이미지 + ffmpeg + Lottie + 슬라이드쇼**로 우회

### 2.2 컨텐츠 형식

| 형식 | 워크플로우 | 출력 길이 |
|------|----------|---------|
| **줌·팬 인트로** | SDXL 이미지 → ffmpeg `zoompan` 효과 | 5~10초 |
| **슬라이드쇼** | 이미지 N장 + 텍스트 오버레이 + BGM(저작권 free) | 15~60초 |
| **Lottie 애니메이션** | LLM이 SVG + CSS/JS 애니메이션 → Lottie 변환 | 2~5초 |
| **인포그래픽 → 모션** | SVG 인포그래픽 → Lottie 트랜지션 | 5~15초 |

### 2.3 운영

- 새 워커: `celery-video` (ffmpeg subprocess)
- 라이선스: BGM은 Pixabay Music, YouTube Audio Library 등 free 명시
- 출력: MP4 (H.264) + WebM
- 다운로드 형태: zip (영상 + 사용된 이미지·BGM 출처 메모)

### 2.4 진입 조건

- Pinterest/Plausible로 영상 수요 데이터 확보
- 일 5~10개 자동 발행 가능 검증

## 3. 트랙 2 — 음향·음악

### 3.1 원칙 — **신중**

- 음악은 라이선스 분쟁 가능성 가장 큼
- Stable Audio 등 모델은 학습 데이터 분쟁 진행 중 (2026년 시점)
- 첫 단계는 **자체 생성보다 큐레이션 + 출처 표기**

### 3.2 단계

1. **단계 1 (안전)**: 무료 BGM 큐레이션 페이지 (사이트의 영상 트랙과 결합)
   - Pixabay Music, YouTube Audio Library, Free Music Archive 출처 명시
   - 우리 사이트는 발견·필터링·미리듣기·다운로드 안내만
   - 실제 다운로드는 원 사이트로 deep link

2. **단계 2 (검토 필요)**: Stable Audio 또는 자체 모델로 짧은 효과음(SFX) 생성
   - 라이선스 명확한 모델만 사용
   - "AI 생성 SFX" 명시

3. **단계 3 (장기)**: 자체 작곡가 외주 오리지널 BGM 제공 (수익 모델 명확해진 후)

## 4. 트랙 3 — 자체 디지털 상품 (프리미엄)

### 4.1 가설

무료 다운로드 데이터로 "이 자산은 유료 프리미엄 버전이 있으면 사겠다"는 신호가 잡히면, 같은 컨셉의 **확장판 (소스 파일 / 다양한 크기 / 컬러 변형 / 라이선스 확장)** 을 디지털 상품으로 판매.

### 4.2 상품 라인 (예시)

| 라인 | 무료 버전 | 프리미엄 버전 (₩4,900~14,900) |
|------|---------|---------------------------|
| 이력서 팩 | 1종 DOCX | 직군별 5종 + Pages·Keynote 변형 + LinkedIn 배너 + 자기소개서 가이드북 |
| PPT 비즈니스 팩 | 단일 5장 PPT | 50장 풀 데크 + 다크/라이트 테마 + 아이콘 100개 |
| SNS 카드 팩 | 5장 | 30장 + Figma 소스 + 폰트 라이선스 가이드 |

### 4.3 결제·전달

- **Lemon Squeezy** (해외 + 국내 가능, MoR 자체 처리 — 부가세·VAT 자동) 또는 **Stripe + 토스페이먼츠** 듀얼
- 결제 → 라이선스 키 발급 → 다운로드 (다운로드 횟수 무제한, 라이선스는 1인 1팀)
- DRM 없음 (사용자 친화) — 영구 라이선스 영수증만 명확

### 4.4 KPI

- 첫 3개월 누적 매출 ₩500,000+
- 환불률 < 5%

## 5. 트랙 4 — B2B 벌크 라이선스 / API

### 5.1 가설

기업이 사내 자료 작성용으로 "직원 100명에게 무제한 다운로드 제공" + "API로 자체 도구에 통합"을 원할 수 있음.

### 5.2 패키지 (예시)

| 플랜 | 가격 | 한도 |
|------|------|------|
| Business | ₩99,000/월 | 시트 5개, 다운로드 무제한 |
| Team | ₩290,000/월 | 시트 25개, API 1만 호출/월 |
| Enterprise | 협의 | 무제한, SLA, 커스텀 모델, 자체 LoRA |

### 5.3 기술

- 워크스페이스 모델 (`workspaces`, `workspace_members`)
- API 키 관리 + Rate limit
- 결제: Stripe Subscription
- SSO (SAML/OIDC) — Enterprise

### 5.4 진입 조건

- 실제 B2B 문의 3건 이상 도착 (문의 폼만 파놓고 데이터 수집)
- 1인 운영자 부담 검토 후 확장 결정

## 6. 트랙 5 — 모바일 앱

### 6.1 가설

모바일 비중이 50%+가 되면 PWA로 충분하지 않은 시나리오(오프라인 다운로드 / 앱 푸시 알림 / 갤러리 통합) 의 가치가 생김.

### 6.2 단계

1. **PWA** (Phase 5 W1~W2): manifest, service worker, 오프라인 캐시
2. **Capacitor** (Phase 5 W3~W6): PWA를 iOS·Android 앱으로 패키징
3. **네이티브** (Phase 5 다음 사이클): React Native 또는 Flutter — 진입 어려움 큼

### 6.3 진입 조건

- MAU 30,000+
- 모바일 사용자 비중 50%+
- 앱스토어 운영 부담 감수 의지 (App Store $99/년, Play $25 일회)

## 7. 인프라 정리 (Phase 5 횡단)

> Phase 5 진입 시점에 main-hub의 25개 컨테이너 + 신규 컨테이너 = 30+개. 인프라 단순화 검토.

### 7.1 옵션 A — Traefik 도입 (D-2 결정 전환)

- 시스템 nginx → Traefik 컨테이너로 교체
- 25개 기존 컨테이너 + 신규 모두 docker label로 자동 라우팅
- 자동 TLS
- **대공사** — Phase 5에서만 검토 (위험 수용 가능)

### 7.2 옵션 B — Kubernetes (k3s)

- 30개+ 컨테이너 + 다중 노드(데스크탑 GPU 노드 등) 가능
- 1인 운영에는 과한 복잡도
- 비추 (단, 본업 떠나서 풀타임 시 검토)

### 7.3 옵션 C — 현 상태 유지 + 정기 청소

- 정기 컨테이너·이미지 prune
- 의존성·이미지 버전 정리
- **추천 (1인 운영 적합)**
