> **목적**: 사이트의 정체성·톤·시각언어를 한 문서에 고정한다. Phase 1 프론트엔드 작업 시 모든 컴포넌트가 이 문서를 baseline으로 삼는다.
> **작성일**: 2026-04-27

## 1. 미션 & 포지셔닝

### 1.1 미션 한 줄

> **"AI로 만들 수 있는 모든 무료 에셋과, 그것을 직접 만드는 방법을 한곳에서."**

### 1.2 포지셔닝 한 단락

> 사무직 직장인이 보고서·제안서·발표자료를 만들 때 가장 빨리 도달하는 무료 자산 사이트가 된다. 단순 다운로드를 넘어 "AI로 어떻게 만들었는지"까지 함께 보여줘서, 다음에는 직접 만들고 싶어지게 한다. AI 컨텐츠지만 사람의 큐레이션과 본업 인사이트가 깃든 콘텐츠 허브.

### 1.3 우리는 ~다 / 우리는 ~가 아니다

| 우리는 ~다 | 우리는 ~가 아니다 |
|-----------|-----------------|
| 직장인의 야근 30분을 줄여주는 도구 | 디자이너 포트폴리오 사이트 |
| AI 생성물임을 당당히 명시하는 사이트 | AI 흔적 지우고 사람 작성 위장 |
| 큐레이션된 자산만 노출 | 무한 자동 발행 쓰레기 더미 |
| 만드는 법까지 알려주는 학습 채널 | 받기만 하는 소비형 사이트 |
| 영구 사용권의 깔끔한 라이선스 | 유료 구독 종속 |
| 한국어 1순위, 다국어 점진 확장 | 영어 우선 글로벌 사이트 |

### 1.4 핵심 사용자 약속 (3가지)

1. **회원가입 후 1분 안에 첫 다운로드 받게 해드립니다.**
2. **모든 자산은 영구 사용 가능하고, 어디에 어떻게 쓰는지 우리가 캐묻지 않습니다.**
3. **AI 생성물임을 숨기지 않고, 그래서 당신이 안전하게 쓸 수 있도록 라이선스를 명확히 해드립니다.**

## 3. 톤 & 보이스

### 3.1 어조 원칙 5

1. **친근한 프로**: "안녕하세요, 박지원 님" 같은 격식은 유지하되, "도움이 되셨길 바랍니다" 같은 거리감 있는 표현은 회피.
2. **명령문 대신 권유문**: "다운로드하세요" 대신 "다운로드해 보세요". 단, 폼 검증 같은 시스템 메시지는 명령형 OK.
3. **존댓말 일관성**: 모든 사용자 대상 텍스트는 존댓말. 어드민·에러 로그·개발자 문서는 평어 가능.
4. **숫자에 단위 명시**: "30회" "5분" "1.2 MB" — 모호한 표현 회피.
5. **이모지 절제**: 기능 텍스트(버튼, 라벨)에 이모지 사용 금지. 빈 상태·축하 메시지엔 1개 정도 OK.

### 3.2 마이크로카피 사전 (자주 등장)

| 상황 | 사용 표현 | 회피 표현 |
|------|----------|----------|
| 다운로드 버튼 | "다운로드" | "받기", "Download" 단독 표기 |
| 검색 결과 0건 | "찾으시는 자료가 아직 없어요. [요청하기]로 알려주시면 우선 만들어볼게요." | "검색 결과가 없습니다." |
| 회원가입 유도 | "1분이면 가입 끝, 영구 사용권 받기" | "지금 회원가입!" |
| 다운로드 성공 | "다운로드 완료. 잘 쓰셨길 바라요." | "성공!" |
| 시간당 한도 초과 | "잠깐 쉬었다 와요. 시간당 30회까지 받을 수 있어요. 다음 다운로드는 N분 뒤부터." | "Rate limit exceeded" |
| 네트워크 오류 | "잠시 연결이 끊겼어요. 다시 시도해 주세요." | "Network Error" |
| 빈 즐겨찾기 | "마음에 드는 자료에 ♡를 누르면 여기 모여요." | "즐겨찾기가 비어있습니다." |
| AI 배지 라벨 | "AI 생성" | "AI Generated" 단독 |
| 라이선스 요약 | "받은 건 평생 자유롭게 쓰셔도 돼요. 단, 이 사이트의 다른 사람에게 그대로 다시 배포하는 것만 안 됩니다." | (장문 법조문 노출) |

### 3.3 카피 시그니처 패턴

- **헤드라인**: "동사 + 명사" 또는 "결과를 약속". 예: "오늘 회의 끝나기 전에 발표자료 완성하세요."
- **서브헤드**: 헤드라인의 보조 설명, 한 줄.
- **CTA**: 동사 단어 1~2개. "지금 다운로드", "회원가입하기".

## 4. 컬러 시스템

### 4.1 컬러 결정 원칙

- **1차색(Primary) 1개 + 보조색(Accent) 1개 + 중립(Neutral) 11단계** 만 사용.
- 1차색은 신뢰·테크 = **인디고 계열**. AI 사이트 표준에 가까워 학습 비용 ↓.
- 보조색은 따뜻한 강조 = **앰버 계열**. 광고/AI 배지/CTA 강조에 사용.
- 무지개색·그라디언트는 **마케팅 페이지 1곳(/about)** 에만 허용.

## 6. 간격·반경·그림자

### 6.1 간격(Spacing)

Tailwind 기본 4px 베이스 사용. 자주 쓰는 상한선만 합의:
- 카드 내부 padding: `p-4` (16px) ~ `p-6` (24px)
- 섹션 사이 마진: `my-12` (48px) ~ `my-20` (80px)
- 그리드 gap: `gap-4` (16px) 데스크탑, `gap-3` (12px) 모바일

### 6.2 라운드(Radius)

- 버튼·입력박스·카드: `rounded-lg` (8px) — Linear와 동일
- 모달·시트: `rounded-xl` (12px)
- 칩·태그·아바타: `rounded-full`
- 이미지 미리보기: `rounded-md` (6px) — 콘텐츠 자체 형태 강조

### 6.3 그림자

- 카드 기본: `shadow-sm` (라이트), 다크에서는 그림자 거의 없고 `border` 강조
- hover: `shadow-md`
- 모달/팝오버: `shadow-xl`
- 다크모드 그림자는 효과 미미 → border 대비로 대체

## 7. 컴포넌트 베이스 (shadcn/ui 위)

### 7.1 채택 원칙

- shadcn/ui v4를 베이스로, 색·간격·라운드만 §4~§6 토큰으로 덮어씀.
- 컴포넌트를 자체 fork — `apps/web/src/components/ui/`에 저장 (npm 의존성 X).
- Headless 동작은 Radix UI primitives 그대로.

### 7.2 자주 쓰는 컴포넌트와 변형

| 컴포넌트 | 변형 | 용도 |
|---------|------|------|
| Button | `variant: primary | secondary | ghost | outline | destructive`, `size: sm | md | lg` | CTA, 보조 액션 |
| Card | `interactive | static`, hover 상태 변화 | 자산 카드, 블로그 카드 |
| Badge | `variant: default | success | warning | error | ai`, `ai`는 amber | 상태·AI 배지 |
| Input | `default | error`, prefix/suffix 지원 | 검색, 폼 |
| Dialog | size `sm | md | lg | full` | 확인, 미리보기 |
| Sheet | `side: right | bottom`, `bottom`은 모바일 큐레이션 인박스 액션 | 모바일 필터, 액션 |
| Tabs | underline 스타일 (Notion/Linear 톤) | 탐색 카테고리 전환 |
| Toast | variant 4종, 우하단(데스크탑)·상단(모바일) | 다운로드 완료, 한도 안내 |
| Tooltip | dark 톤 고정 (라이트/다크 모두 어두운 배경) | 단축키 안내, 라벨 보충 |
| Skeleton | shimmer 애니메이션 | 로딩 상태 |
| EmptyState | 일러스트 + 메시지 + 1차 CTA | 검색 0건, 즐겨찾기 비음 |

### 7.3 Phase별 컴포넌트 우선순위

| Phase | 우선 컴포넌트 |
|:---:|------|
| 1 | Button, Card, Badge, Input, Dialog, Toast, Skeleton, EmptyState |
| 1 | AssetCard(자산 카드 — Card 위에 빌드), MasonryGrid, Pagination, FilterChips |
| 1 (admin) | DataTable, KeyboardShortcutBar (큐레이션 인박스) |
| 2 | LangSwitcher, OAuthButton, FavoriteButton |
| 3 | BlogCard, ArticleLayout, AuthorBlock, NewsletterSignup |

## 9. 일러스트 정책

### 9.1 사용 시점

- **빈 상태(Empty State)**: 검색 0건, 즐겨찾기 비음, 다운로드 내역 없음 (Phase 1부터)

- **에러 페이지**: 404, 500 (Phase 1)
- **About / 랜딩 히어로**: 사이트 미션 시각화 (Phase 1 후반)

### 9.2 출처 (라이선스 무관 무료)

### 9.3 스타일 통일 룰

## 10. 이미지·미디어 가이드

### 10.1 자산 미리보기

- 그리드 카드: 비율 유지 마소닉 (2:3 ~ 3:2 다양)
- 상세 페이지: 가로 최대 1200px, 비율 유지, 클릭 시 라이트박스
- 포맷: WebP 우선, AVIF는 Phase 2에서 검토
- 워터마크: 미리보기에 작은 "AI 생성" 배지 코너 (다운로드 파일에는 워터마크 X, 메타데이터로만)

### 10.2 카드 hover 상태

- hover 시 카드 위로 살짝 떠오름(`translate-y-[-2px]`) + `shadow-md`
- 우상단에 ♡ 즐겨찾기, 다운로드 카운트 노출
- 모바일은 hover 없음 — 탭 시 상세 페이지로 즉시 이동

### 10.3 AI 생성 배지

- 모든 자산 카드 좌하단에 작은 chip:
  ```
  [✨ AI] (amber-500 outline + amber-50 bg)
  ```
- 자산 상세 페이지 상단에 큰 배지:
  ```
  [✨ AI 생성 콘텐츠] [모델: Flux 1.0 dev] [생성일: 2026-04-30]
  ```

## 11. 애니메이션·모션

### 11.1 원칙

- **목적 있는 모션만**. 장식용 애니메이션 금지.
- 지속시간: 80~250ms. 페이지 전환은 0ms (즉시).
- easing: `ease-out` 기본, modal/sheet 등장은 `cubic-bezier(0.16, 1, 0.3, 1)` (Linear의 spring 톤).
- `prefers-reduced-motion: reduce` 존중 — 모든 애니메이션 비활성화.

### 11.2 표준 트랜지션

| 상황 | 효과 |
|------|------|
| 버튼 hover | 100ms color/bg 트랜지션 |
| 카드 hover | 150ms transform + shadow |
| 모달 등장 | 200ms scale(0.96 → 1) + opacity |
| Toast 등장 | 250ms slide-up + opacity |
| 페이지 라우트 변경 | 없음 (Next.js prefetch + skeleton로 체감 빠르게) |

| 토큰 | 값 | 우선 페이지 |
|------|----|----------|
| `sm` | 640px | 모바일 가로 |

- 컬러: 기존 인디고 중심 유지, 앰버는 작은 액션/강조에만 사용
- 워드마크: 화면 표시명은 `Vaultix`, 파일/패키지/컨테이너는 `vaultix`

### 15.3 워드마크 가이드

- 정자체로 유지, 제품명 표기는 `Vaultix`를 기본으로 한다.
- 옵션: `a`의 도트 또는 `i`의 도트를 brand.accent(amber)로 1픽셀 강조 (브랜드 시그니처).
