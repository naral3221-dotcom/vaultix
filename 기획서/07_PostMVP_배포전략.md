# 07 — MVP 이후 배포 전략

> **목적**: MVP가 끝난 뒤 다음 버전으로 빠르게 이어가기 위한 릴리즈 순서.
> **기준**: v0.4 구현 단일본 완료 이후 적용.
> **작성일**: 2026-04-29

---

## 1. 배포 원칙

1. **한 번에 한 축만 넓힌다.** 언어, 콘텐츠 타입, SEO, 수익화를 동시에 열지 않는다.
2. **운영 대시보드를 먼저 본다.** 다운로드율, 신고, 생성 실패율, 검색 유입을 보고 다음 배포를 고른다.
3. **MVP의 안정성을 해치지 않는다.** 새 기능은 feature flag로 감싸고, panic에서 끌 수 있게 한다.
4. **자동화보다 검증이 먼저다.** 자동 발행량을 늘리기 전에 품질 메트릭과 신고 흐름이 안정적인지 확인한다.
5. **문서는 배포와 같이 갱신한다.** 새 결정은 `00_확정사항_레지스트리.md`에 ID를 추가한다.

---

## 2. Release 1 — MVP 안정화

**목표**: 실제 사용자 10~30명이 써도 깨지지 않는 이미지 사이트.

### 포함

- 베타 피드백 반영
- 검색/필터 UX 다듬기
- 다운로드 모달 마찰 제거
- `/today`, `/log` 본문 운영 시작
- 뉴스레터 첫 정기 발송
- 자산 품질 대시보드 Top/Worst 10 추가
- 신고 처리 SOP 1회 리허설

### 완료 조건

- 베타 사용자 10명 이상
- 다운로드 성공률 95% 이상
- 신고/문의 응답 SLA 1회 검증
- 생성 실패율 10% 미만
- 운영자가 매일 봐야 하는 대시보드 위젯 10개 이하로 정리

---

## 3. Release 2 — 일본어 출시

**목표**: 한국어 이미지 자산을 일본어 검색/사용 흐름으로 확장.

### 포함

- next-intl locale `ko`, `ja`
- UI 일본어 수동 번역
- 라이선스/Privacy/Terms 일본어 본문
- DeepL ko→ja 메타데이터 번역
- hreflang, 언어별 sitemap
- 일본어 카테고리/태그 라벨
- 일본어 Search Console 관찰

### 제외

- 영어 출시
- PPT/SVG/DOCX 생성
- 대규모 블로그 자동화

### 완료 조건

- 일본어 UI 누락 키 0개
- 일본어 자산 메타 1,000건 이상
- 일본어 자산 상세/다운로드 E2E 통과
- 일본어 sitemap 제출 완료

---

## 4. Release 3 — 레시피 공개와 ComfyUI 특수 워크플로우

**목표**: 단순 다운로드 사이트가 아니라 "만드는 법까지 공개하는 AI 자산 허브"로 차별화.

### 포함

- `asset_recipes`
- 자산 상세 레시피 섹션
- 프롬프트/시드/모델/생성 파라미터 공개
- ComfyUI 워크플로우 JSON 다운로드
- `consistent_character`, `controlnet_pose` 2종 특수 워크플로우
- 인박스에서 "시리즈로 만들기", "포즈 변형" 트리거

### 완료 조건

- 공개 자산 500건 이상에 레시피 연결
- 레시피 복사/다운로드 이벤트 추적
- 특수 워크플로우 2종이 인박스에서 정상 생성
- 신고된 레시피가 `blocked_recipes`로 재생성 차단됨

---

## 5. Release 4 — 콘텐츠 타입 확장 1차

**목표**: 이미지 사이트에서 실무형 자산 허브로 확장.

### 우선순위

1. PPT 템플릿
2. SVG 인포그래픽
3. DOCX 이력서/자소서

### 포함

- content_type별 워커 분리
- LibreOffice headless 미리보기
- 타입별 인박스 미리보기 컴포넌트
- 다운로드 파일 묶음 정책
- 타입별 품질 점수 기준

### 완료 조건

- 이미지 외 타입 3종 중 최소 1종 공개
- 해당 타입 자산 100개 이상
- 미리보기 렌더 실패율 5% 미만
- 다운로드/라이선스/레시피 흐름이 이미지와 같은 패턴으로 동작

---

## 6. Release 5 — 메타 콘텐츠와 AdSense 준비

**목표**: 검색과 광고 승인을 위한 사람 손이 보이는 콘텐츠 층 만들기.

### 포함

- 블로그/가이드 IA 확정
- 첫 10편 수동+LLM 보조 작성
- 자산↔가이드 임베딩 추천
- long-tail 키워드 200개 매핑
- 내부 링크 자동화
- AdSense 정책 사전 점검

### 완료 조건

- 블로그/가이드 10편 공개
- 자산 상세에서 관련 가이드 3개 추천
- Search Console 인덱싱 페이지 300개 이상
- 광고 슬롯은 자리만 두고 실제 광고는 승인 후 활성화

---

## 7. Release 6 — AdSense 신청과 수익화 1차

**목표**: 광고 수익화를 시작하되 다운로드 UX를 해치지 않는다.

### 포함

- AdSense 신청
- 승인 후 `AdSlot` 활성화
- 광고와 다운로드 버튼 최소 거리 유지
- 네이티브 광고 카드 시각 구분 강화
- 제휴 링크는 도구 리뷰 글에만 제한적으로 배치

### 완료 조건

- AdSense 승인 또는 거절 사유 기록
- 광고로 인한 Core Web Vitals 악화 없음
- 다운로드 전환율 10% 이상 하락 시 광고 위치 롤백

## 10. 다음 개발 진행 방식

개발 에이전트에게는 매번 전체 기획서를 다시 주지 않는다.

### 새 작업 시작 입력

1. `00_IMPLEMENTATION_SPEC_v0.4.md`
2. 현재 릴리즈 섹션만 발췌
3. 관련 A/B 문서
4. 직전 배포 회고

### 작업 단위

- DB 변경 1개
- API 묶음 1개
- UI 화면 1개
- 워커 1개
- E2E 시나리오 1개

이 단위로 PR 또는 커밋을 나눈다. 기능이 커지면 다음 릴리즈로 넘긴다.
