## 1. 사이트 미션 (v0.2 계승)

> **"AI로 만들 수 있는 모든 무료 에셋과, 그것을 직접 만드는 방법을 한곳에서."**

- **단순 다운로드 사이트가 아닌 "AI 생산성 허브"**
- 사용자 동기 두 가지:
  1. 즉시 받아 쓰기 (이미지·PPT·템플릿)
  2. 만드는 법 배우기 (가이드·프롬프트·도구 리뷰)

### 1.1 비즈니스 모델

| 우선순위 | 수익원 | 비고 |
|:---:|--------|------|
| 1 | Google AdSense | 손익분기 도달 핵심 |
| 2 | 제휴 마케팅 (쿠팡파트너스, AI 도구 리뷰) | 자연스러운 수익화 |
| 3 | 디지털 상품 (프리미엄 템플릿) | 장기 |
| 4 | 뉴스레터 스폰서 | 트래픽 확보 후 |
| 5 | B2B 벌크 라이선스 / API | 기업 대상 (장기) |

### 1.2 핵심 USP

1. **다양성 압도** — 이미지·PPT·인포그래픽·템플릿·코드를 한 곳에서
2. **무료 + 영구 라이선스** — 받은 건 평생 사용 가능
3. **AI 활용 가이드 동시 제공** — 받기만 하지 않고 만드는 법까지
4. **다국어** — 한·영·일·중·스페인어·베트남어 점진 확대
5. **상업적 사용 안전 + 책임 한계 명시** — 라이선스 모호함 해소
6. **AI 생성 콘텐츠 사이트지만 메타-콘텐츠로 정체성** — AdSense 친화

### 2.2 후순위 / 별도 처리

- **음향/음악**: Stable Audio 등 모델 검토 필요. 라이선스 까다로움. Phase 4+
- **3D 모델 / 폰트**: 워크플로우 복잡. 검토 시점 미정

### 2.3 영상 컨텐츠 정책 (신규 — JH님 결정 반영)

**원칙**: 영상 생성 API(Runway, Pika, Sora 등) 사용 X. 비용 폭탄 위험.

**대안 워크플로우**:

1. **이미지 + 모션 합성 (저비용)**
   - SDXL로 이미지 생성 → ffmpeg로 줌/팬/페이드 효과 추가
   - 5초 인트로/아웃트로, 짧은 트랜지션 클립 등
   - 비용: 거의 0 (로컬 GPU + ffmpeg)

2. **Lottie / SVG 애니메이션**
   - LLM이 SVG 코드 + CSS/JS 애니메이션 생성
   - 웹 / 앱 UI에 활용 가능한 가벼운 모션
   - 다운로드 형태: JSON (Lottie) 또는 HTML/SVG

3. **슬라이드쇼 영상**
   - 이미지 N장 + 텍스트 오버레이 + BGM (저작권 free) → ffmpeg 자동 합성
   - 인스타 릴스, 틱톡, 유튜브 쇼츠용

4. **Tier 검토 시점**
   - Phase 4 이후, Tier 1+2가 안정화된 다음에 결정
   - 다운로드 데이터를 보고 시장 수요 판단

### 2.4 메타 컨텐츠 (블로그/가이드 — 자동화 파이프라인은 §4에서 별도)

- 도구 리뷰 / 사용 가이드 / 사례 연구
- AdSense 승인의 핵심 자산
- JH님 본업(광고 운영) 인사이트 결합
- 발행 목표: 주 2~3편, 6개월 후 100편 누적

## 3. 컨텐츠 자동화 파이프라인 (v0.2 강화)

### 3.1 전체 아키텍처

```
┌─────────────────────────────────────────────────────────┐
│              [ 프롬프트 풀 (DB) ]                        │
│   카테고리 × 스타일 × 변수 = 수만 가지 조합              │
│   예: 이미지(9만개), PPT(5천개), 템플릿(2천개) ...        │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│              [ 컨텐츠 생성 큐 (Celery) ]                 │
│   타입별 워커 분리:                                     │
│   image / pptx / svg / docx / html / xlsx / lottie      │
└─────────────────────────────────────────────────────────┘
                         ↓
        ┌────────┬────────┬────────┬────────┬────────┐
        ↓        ↓        ↓        ↓        ↓        ↓
   [이미지]  [PPT]   [SVG]   [DOCX]  [XLSX]  [HTML]
   SDXL     pptx+LLM 코드+검증 docx   xlsx    LLM+빌드
        ↓        ↓        ↓        ↓        ↓        ↓
┌─────────────────────────────────────────────────────────┐
│        [ 1단계: 자동 검증 (타입별) ]                     │
│   이미지: 블러/NSFW/유사도   PPT: 무결성/렌더              │
│   SVG: lint/렌더 OK         DOCX: 스키마                 │
│   XLSX: 수식 동작            HTML: 빌드 성공              │
└─────────────────────────────────────────────────────────┘
                         ↓ (통과)
┌─────────────────────────────────────────────────────────┐
│        [ 2단계: AI 품질 평가 (LLM/CLIP 점수) ]           │
│   - 미적 점수, 카테고리 적합도, 중복도                    │
│   - 점수 0.4 미만: 자동 폐기                             │
│   - 점수 0.4~0.7: 큐레이션 인박스                        │
│   - 점수 0.7+: 기본 승인 상태로 인박스                    │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│        [ 3단계: 메타데이터 자동 생성 ]                   │
│   - 제목/설명/태그/alt텍스트 (로컬 LLM 또는 API)         │
│   - 다국어 번역 (DeepL)                                 │
│   - 카테고리 자동 분류 (CLIP/임베딩)                    │
│   - SEO 키워드 매핑                                     │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│        [ 4단계: 큐레이션 인박스 (관리자) ]               │
│   - 카드 그리드, 키보드 단축키 중심                      │
│   - 일 30분 / 100~150건 처리                            │
└─────────────────────────────────────────────────────────┘
                         ↓ (승인)
┌─────────────────────────────────────────────────────────┐
│        [ 5단계: 발행 큐 (시간 분산) ]                    │
│   - 시간당 N건씩 / 카테고리·언어 균형                    │
└─────────────────────────────────────────────────────────┘
                         ↓
                    [ 사이트 노출 ]
```

### 3.2 컨텐츠 타입별 어댑터 패턴

```python
class ContentGenerator(Protocol):
    content_type: str  # "image" | "pptx" | "svg" | "docx" | "xlsx" | "html"

    def generate(self, prompt: Prompt) -> RawContent: ...
    def validate(self, raw: RawContent) -> ValidationResult: ...
    def score(self, raw: RawContent) -> float: ...
    def extract_metadata(self, raw: RawContent) -> dict: ...
    def render_preview(self, raw: RawContent) -> bytes: ...
```

각 타입별 구현:
- `ImageGenerator`: Flux/SDXL → PIL → WebP
- `PptxGenerator`: 로컬 LLM이 슬라이드 구조 JSON → python-pptx → LibreOffice headless로 PNG 미리보기
- `SvgGenerator`: LLM → svgo lint → 브라우저 렌더 검증
- `DocxGenerator`: LLM 구조 → python-docx → PDF 미리보기
- `XlsxGenerator`: LLM 시나리오 → openpyxl → 수식 검증
- `HtmlGenerator`: LLM → Tailwind 빌드 → Puppeteer 스크린샷
- `LottieGenerator`: LLM JSON 생성 → 렌더 검증

### 3.3 프롬프트 풀 시스템

```sql
prompt_templates (
  id, content_type, category_id,
  base_prompt,        -- "{subject}, {style}, {mood}, professional"
  variables_json,     -- {"subject": [...], "style": [...]}
  active, weight,
  performance_score   -- 발행 후 다운로드 수 기반 자동 가중치
)
```

- 카테고리별 프롬프트 템플릿 시드 데이터 작성 (Phase 0)
- 발행 후 다운로드 데이터로 자동 가중치 조정 (Phase 3+)
- 인기 카테고리 자동 강화, 비인기 자동 약화

### 3.4 큐레이션 인박스 UX

**키보드 중심 조작**:
- `→` 승인 + 다음
- `←` 폐기 + 다음
- `↑` 보류
- `1~9` 카테고리 재지정
- `Space` 메타데이터 편집 모달
- `T` 다국어 번역 재생성
- `?` 도움말

**일일 목표**: 30분 이내, 100~150건 처리

### 4.2 시간 부담

- 주간 토픽 큐레이션: 5분
- 본업 인사이트 추가 (글당): 10~20분 (음성 메모 활용 시 5분)
- 최종 검수 (글당): 10분
- **주 2편 발행 시 총 시간: 약 50~70분/주**

### 4.3 토픽 자동 발굴 시스템 상세

```python
# 매주 월요일 자동 실행
def discover_topics():
    sources = [
        google_trends_korea(),
        google_trends_global(),
        scrape_competitor_rss(),
        analyze_internal_search_logs(),
        analyze_failed_searches(),  # 사용자가 찾지만 결과 없는 검색어
    ]

    raw_topics = aggregate(sources)

    # LLM이 점수화
    scored = llm_score_topics(raw_topics, criteria=[
        "검색량 vs 경쟁도",
        "JH 본업과 연관성",
        "AdSense 정책 친화도",
        "기존 콘텐츠와 중복 여부",
    ])

    # 상위 20개를 슬랙/디스코드로 푸시
    push_to_jh(scored.top(20))
```

### 4.4 콘텐츠 카테고리 (메타 컨텐츠)

## 5. LLM 사용 정책 (v0.3 신규 — 명확화)

### 5.1 원칙

> **LLM은 백엔드 컨텐츠 생성에만 사용한다. 사용자 노출형 LLM 서비스는 제공하지 않는다.**

### 5.2 허용되는 LLM 사용

| 용도 | 사용 위치 | LLM 종류 |
|------|----------|---------|
| 컨텐츠 생성 (PPT, DOCX, SVG 등) | 백엔드 워커 | 로컬 (Ollama) 우선, 필요 시 API |
| 메타데이터 생성 (제목/설명/태그) | 백엔드 | 로컬 LLM 우선 |
| 다국어 번역 | 백엔드 | DeepL API (LLM보다 품질 좋음) |
| 품질 점수 평가 | 백엔드 | CLIP / 로컬 LLM |
| 블로그 초안 작성 | 백엔드 | API 가능 (질 우선) |
| 토픽 발굴 | 백엔드 | 로컬 LLM 또는 API |
| 카테고리 자동 분류 | 백엔드 | 임베딩 모델 (로컬) |
| 라이선스 검토 (개발 단계) | 외부 (Claude/GPT) | 일회성 |

### 5.3 금지되는 LLM 사용

- ❌ 사용자가 프롬프트 입력해서 실시간 이미지 생성
- ❌ 사용자가 채팅으로 도와달라고 하는 챗봇
- ❌ 사용자별 맞춤 추천 (LLM 기반)
- ❌ 어떤 형태든 사용자 입력 → LLM → 응답 구조

### 5.4 LLM 호스팅 전략

**1차: 로컬 LLM (Ollama)**
- 모델 후보: Llama 3.1 70B (메타데이터·번역), Qwen 2.5 (한국어), Mistral
- 장점: 비용 0, 데이터 외부 유출 X
- 단점: GPU 점유 (이미지 생성과 시간 분리 필요)

**2차: 외부 API (필요 시)**
- 블로그 초안처럼 품질이 매우 중요한 작업
- Claude API, OpenAI API
- 일일 토큰 한도 설정 (예: 100K tokens/day)
- 월 예산 캡 설정 (예: $50)

### 7.2 LLM 기반 라이선스 검토 워크플로우 (v0.3 신규)

> JH님 결정: 변호사 비용 부담 → LLM(Claude/GPT)으로 검토

**전제 한계 인지**:
- LLM은 변호사를 대체하지 못함
- 한국법은 학습 데이터에 영문법보다 적게 포함되어 있어 한국법 부분은 추가 검증 필요
- 그러나 1차 검토, 다른 사이트 비교, 누락 조항 발견에는 충분히 유용

**검토 워크플로우**:

```
[1단계] 레퍼런스 라이선스 수집
   - Pixabay Content License
   - Unsplash License
   - Pexels License
   - Freepik 무료 라이선스
   - Creative Commons (CC BY, CC0)
   - Adobe Stock 라이선스
   → 모두 텍스트로 수집

[2단계] LLM에 비교 검토 요청
   프롬프트 예시:
   "다음은 우리 사이트 라이선스 초안입니다. [초안 첨부]
    그리고 다음은 비슷한 무료 자산 사이트들의 라이선스입니다. [레퍼런스 첨부]

    다음 관점에서 검토해주세요:
    1. 우리 초안에 누락된 중요 조항이 있는가?
    2. 우리 초안에 모호하거나 해석 분쟁 여지가 있는 조항은?
    3. AI 생성물 특수성(저작권, 인격권, 학습 데이터)에서 추가로 명시해야 할 점은?
    4. 한국 법원에서 면책 조항이 인정되기 위한 표현 방식은?
    5. 사용자의 가독성(쉽게 이해할 수 있는가) 관점의 개선점은?"

[3단계] 한국법 특화 검토
   - 한국 소비자보호법: "사업자 면책 조항이 무제한으로 인정되지 않음"
   - 한국 약관규제법: "고객에게 부당하게 불리한 조항 무효"
   - 한국 저작권법 (AI 생성물 관련 판례 동향)
   → 별도로 LLM에 한국법 관점에서 추가 검토 요청
   → 정부 무료 법률상담 활용 (서울시 무료법률상담, 대한법률구조공단)

[4단계] 사용자 친화 버전 작성
   - 법적 정확성 버전 ("법적 텍스트")
   - 쉬운 요약 버전 ("Plain English / Plain Korean")
   - 사이트에는 "쉬운 요약" 표시 + "전체 라이선스 보기" 토글

[5단계] 정기 재검토
   - 분기 1회 LLM에게 "최근 6개월간 한국 AI 생성물 관련 판례·법령 변경 사항"
   - 변경 시 라이선스 v업데이트
```

**LLM 검토 시 주의**:
- 단일 LLM(예: Claude만)이 아니라 두 가지 이상 LLM(Claude + GPT)에 같은 질문 → 답변 차이 비교
- LLM이 "확실히 안전합니다"라고 단언하면 의심 (LLM은 법률 자문 자격 없음)
- 큰 분쟁 가능성이 보이는 조항은 무료 법률상담 활용

### 7.3 사이트 내 AI 생성 명시

async def download_asset(user, asset):
    if not user.email_verified: raise 403
    if redis.get(f"dl:{user.id}:{hour}") >= 30: raise 429

    prior = db.query(Download).filter_by(user_id=user.id, asset_id=asset.id).first()
    signed_url = generate_signed_url(asset.file_path, ttl=300)

## 부록 C. 컨텐츠 자동화 파이프라인 요약 (한눈에 보기)

### 자산 생성 파이프라인 (이미지/PPT/SVG/DOCX/...)
```
프롬프트 풀 → 타입별 워커 → 자동 검증 → AI 점수 →
메타데이터 자동 생성 → 큐레이션 인박스(30분) → 시간 분산 발행
```

### 메타 콘텐츠 파이프라인 (블로그)
```
주간 토픽 발굴 → JH 큐레이션(5분) → 자료 수집 + 초안 자동 작성 →
JH 인사이트 추가(음성 5분 또는 메모 10분) → LLM 통합 →
JH 최종 검수(10분) → 다국어 자동 번역 → 발행
```

### 1주일 운영 시간 추정
- 큐레이션 인박스: 30분 × 7일 = 3.5시간
- 토픽 큐레이션: 5분 × 1회 = 5분
- 블로그 인사이트 + 검수: 30분 × 2편 = 1시간
- 뉴스레터 검수: 30분
- **총합: 약 5~6시간/주**

본업 외 시간으로 충분히 운영 가능한 수준.
